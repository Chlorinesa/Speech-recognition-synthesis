# Speech Recognition & Synthesis Web Application

Веб-приложение для синтеза речи (Text-to-Speech) и распознавания речи (Speech-to-Text) с возможностью сохранения настроек и истории операций.

## Возможности

### Синтез речи (TTS)
- Преобразование текста в речь через Web Speech API
- Выбор голоса из доступных в системе
- Настройка скорости, высоты тона и громкости
- Визуальная подсветка текущего произносимого текста
- Настройка цветов подсветки
- Управление воспроизведением: старт, пауза, возобновление, остановка

### Распознавание речи (STT)
- Загрузка аудиофайлов через drag-and-drop (MP3, WAV, M4A, FLAC)
- Извлечение текста через внешний сервер
- Отображение времени обработки
- Копирование результата и сохранение в TXT
- Отмена процесса распознавания

### Пользовательская система
- Регистрация и авторизация
- Сохранение настроек синтеза речи
- История всех операций
- Темная/светлая тема интерфейса
- Адаптивный дизайн

## Установка и запуск

### Предварительные требования
- Open Server Panel
- PostgreSQL 12+
- Node.js 16+ и npm

### 1. Клонирование репозитория
```bash
git clone https://github.com/Chlorinesa/Speech-recognition-synthesis.git
cd Speech-recognition-synthesis
```

### 2. Настройка базы данных
1. Создайте базу данных в PostgreSQL:
```sql
CREATE DATABASE synthesis_db;
```

2. Импортируйте схему базы данных:
```bash
psql -U postgres -d synthesis_db -f script.sql
```

### 3. Настройка переменных окружения
1. Скопируйте файл `.env.example` в `.env`:
```bash
cp .env.example .env
```

2. Отредактируйте файл `.env` и укажите свои данные для подключения к БД:
```env
DB_HOST=localhost
DB_NAME=synthesis_db
DB_USER=postgres
DB_PASSWORD=your_password_here
```

### 4. Настройка React приложения
```bash
# Установите зависимости
npm install

# Для разработки
npm run dev

# Для продакшена
npm run build
```

### 5. Запуск Open Server Panel
- Запустите Apache, PHP 7.4+ и PostgreSQL в Open Server Panel
- Поместите проект в папку `domains` или настройте виртуальный хост

### 6. Доступ к приложению
- Разработка: `http://localhost:5173`
- Продакшн: `http://localhost/your-project-folder`

## Конфигурация

### Настройка сервера распознавания (STT)
По умолчанию приложение использует сервер по адресу `http://172.31.1.1:5002`.
Для изменения адреса обновите файл `src/hooks/useSttService.js`.

### Настройка базы данных
Все параметры подключения к базе данных хранятся в файле `.env`. Файл `php/db_connection.php` автоматически читает настройки из этого файла.

## Безопасность
- Пароли пользователей хранятся в хэшированном виде с использованием BCrypt
- Защита от SQL-инъекций через параметризованные запросы
- Валидация всех входных данных на стороне клиента и сервера
- Сессионное управление аутентификацией
- Конфиденциальные данные (пароли БД) хранятся в файле `.env`, который не коммитится в репозиторий

## Устранение неполадок

### Ошибка подключения к базе данных
1. Проверьте, запущен ли PostgreSQL
2. Убедитесь, что файл `.env` существует и содержит правильные параметры
3. Проверьте логи PostgreSQL на наличие ошибок подключения

### Web Speech API не поддерживается
Используйте современные версии браузеров:
- Chrome 33+
- Edge 79+
- Safari 14+

### Сервер распознавания недоступен
Убедитесь, что STT сервер запущен и доступен по адресу `http://172.31.1.1:5002`. При необходимости измените адрес в `src/hooks/useSttService.js`.
